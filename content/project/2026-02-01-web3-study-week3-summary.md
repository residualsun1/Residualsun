第三周的时间都用于准备休闲黑客松，基本没有专门学习 Web3 领域的知识——突然想起前几天 wachi 老师在群里分享了一位学员的打卡笔记，上面很简短的写了两个字：「在松」。哈哈，大家看了都觉得很搞笑，现在写第三周学习总结回想起来时也一样觉得很搞笑。

不打一次休闲黑客松，我也将无从得知黑客松的魅力。什么是黑客松？简单来说就是指一种编程马拉松活动，由开发者、设计师、产品经理等不同背景的专业人士在限定时间内——通常 24 至 72 小时——协作完成一个可演示的软件或硬件原型的竞赛活动，核心特点是高强度、团队合作和创新，旨在快速将创意转化为实际产品。

总的来说，我在这一次黑客松中的收获可以被归纳为以下几点：

第一，开始迈向 Vibe Coding。不论是实习计划，还是黑客松，我给自己的定位一直是技术向，因而在自己缺乏编程经验的情况下，学会 Vibe Coding 是最低限度的要求。起初我是通过 Antigravity 这样的 IDE 调用 Claude Opus 4.5 等大模型来进行 Vibe Coding，但由于提示词使用不当，也不懂得构建自己的 Vibe 工作流，许多 Tokens 轻易便被浪费。为确保可以继续在 Vibe Coding 中学习，我了解到了中转站的说法，并将 Claude Code 和 Codex 都安装到了自己 Windows 的终端之中，大大增加了我进行 Vibe Coding 的容错率和频率。现在我也算是一个 Vibe Coding 的重度使用用户了，而在此次黑客松中，虽然产品的主要代码主要由小白编写，但我也提供了许多修改建议。

第二，体验到了产品经理的角色。由于我们团队此次黑客松中的代码编写任务主要由小白承担，而我们又不知道两个人同时都进行开发要怎么协调代码，因此我担任起了产品经理这一角色。即便我一开始并不清楚自己究竟在做什么，但看着小白一个人闷着头写代码，我认为自己需要调动队伍的气氛，跟进项目的进度，主动向三位队友发起了三次会议，每次会议我都对小白搭建的网站提供建议——需要增加或减少什么功能、UI 界面如何美化、产品叙事应该如何表述……也正是在我主动开启交流的情况下，小白、ChiliChili 和小马也才相互讨论起来，想出了许多有意思的点子，例如正是在讨论中让小白提出了「时间赌徒」的概念，小马后来基于愿景则提出了第三方对赌的一种模式：老板和员工之间进行对赌。此外，我发现相对于其他几位队友，我更执着于产品应该长什么样子，因此总是禁不住向小白提出建议，认为网站还有什么地方优化。如今回想起来，似乎不知觉间扮演了产品经理的角色，这种感觉在后来我提议小白在网站界面中删减动漫角色而小白不舍得时犹为明显。后来，我一直在心中思考着如何向外界表述产品的叙事，在 Demo Day 临近时下意识地准备路演稿子和 PPT，默认要自己上场路演，否则难以安心。

第三，体验到团队合作的激励感、归属感。或许可以毫不夸张地说，在实习计划的这三周里，我体验最为美好的时刻都存在于那些主动向队友发起会议的时间，只有在一来一回的交流之中我才感觉自己在真正思考，也能感受到队友的输出，其间便孕育着新的想法。当新想法产生以后，大家都激动而更加专注，这种感觉如同心流，却又带有昂扬迈进的心理体验，我们为此兴奋、满足。敲定想法后，小白总是二话不说，埋头就是吭哧吭哧地敲代码，我则是排查和调试 bug，以及根据自己对产品理念、叙事的理解思考具体的功能设计。最后在准备路演彩排时，心中便油然产生共同奋斗的感觉——这种感觉难以形容，或许可以用实习计划中学员的表述来窥探一二：黑客松都是要熬夜写代码的，从这个角度来看，你我之间何尝不是一种过命的交情？

第四，还是 wachi 老师说的 Credit。我很庆幸主动找上小白组队，相比于我，小白在技术学习上的执行力更强，他想到便去做，吭哧吭哧一通便能给你整出东西。但我不行，很容易在跑代码时遇到 Token 不足、bug 频出后便开始拖延，如果要让我自己做一项黑客松产品，那定然不行。所幸小白在技术实现这一板块展现出惊人的执行力，我只需要向他提供改进建议，以及准备好产品路演即可——当然，我在其中还是有参与到部分代码的开发。不过重点是，我之所以选择小白，正是因为在他平时的发言、分享中看出他是一个**值得信赖、靠谱**的人。我记得在一次分享会上，一位嘉宾说到 Web3 是一个极其重视口碑的行业，但现在我的理解是任何行业都是如此，无论在哪，一个人的口碑总是先于他本身，并且是其能力被看见的前提。回顾黑客松的体验，意识到能遇到小白这样的队友源于我对其 credit 的感受，这更让我重视起自身的 credit 建设。
